Methods in Java

Introduction to Methods

A method in Java is a block of code that performs a specific task and can be called when needed. Methods promote code reusability, modularity, and maintainability.

Types of Methods

Java methods can be categorized as:

  • Void Methods: Perform an action without returning a value.
  • Return Methods: Return a value of a specific type after execution.
  • Static Methods: Belong to the class, not an instance, and can be called without creating an object.
  • Instance Methods: Belong to an instance of a class and require an object to be called.

Defining a Method

A method is defined with an access modifier, return type, name, parameters (optional), and body.

// Void method
void sayHello() {
System.out.println("Hello, World!");
}

// Return method
int add(int a, int b) {
return a + b;
}

Access Modifiers

Access modifiers control the visibility of a method:

  • public: Accessible from everywhere.
  • private: Accessible only within the same class.
  • protected: Accessible within the same package and in subclasses.
  • default (no modifier): Accessible within the same package.
public class AccessModifierExample {
public static void main(String[] args) {
publicMethod();
privateMethod();
}

public static void publicMethod() {
System.out.println("This is a public method.");
}

private static void privateMethod() {
System.out.println("This is a private method.");
}
}

Output

This is a public method.
This is a private method.

Void Method Example

A void method performs an action without returning a value.

public class VoidMethodExample {
public static void main(String[] args) {
printGreeting();
}

static void printGreeting() {
System.out.println("Welcome to Java Programming!");
}
}

Output

Welcome to Java Programming!

Return Method Example

A return method computes a value and returns it to the caller.

public class ReturnMethodExample {
public static void main(String[] args) {
int result = multiply(5, 3);
System.out.println("Result: " + result);
}

static int multiply(int a, int b) {
return a * b;
}
}

Output

Result: 15

Method with Parameters

Methods can accept parameters to work with dynamic input.

public class ParameterMethodExample {
public static void main(String[] args) {
greetUser("Alice");
}

static void greetUser(String name) {
System.out.println("Hello, " + name + "!");
}
}

Output

Hello, Alice!

Method Overloading

Method overloading allows multiple methods with the same name but different parameter lists (number or types).

public class OverloadingExample {
public static void main(String[] args) {
System.out.println("Sum (int): " + add(2, 3));
System.out.println("Sum (double): " + add(2.5, 3.5));
}

static int add(int a, int b) {
return a + b;
}

static double add(double a, double b) {
return a + b;
}
}

Output

Sum (int): 5
Sum (double): 6.0

Method Overriding

Method overriding occurs when a subclass provides a specific implementation of a method already defined in its superclass. The method must have the same name, return type, and parameter list.

  • Requires inheritance (a subclass extending a superclass).
  • Use the @Override annotation to indicate overriding.
  • The overridden method must have the same signature and compatible return type.
class Animal {
void makeSound() {
System.out.println("Generic animal sound");
}
}

class Dog extends Animal {
@Override
void makeSound() {
System.out.println("Bark");
}
}

public class OverridingExample {
public static void main(String[] args) {
Animal animal = new Animal();
animal.makeSound();
Animal dog = new Dog(); // Polymorphism
dog.makeSound();
}
}

Output

Generic animal sound
Bark

Static vs. Instance Methods

Static methods belong to the class and can be called without an object. Instance methods require an object of the class.

public class StaticVsInstance {
public static void main(String[] args) {
staticMethod(); // Call static method directly
StaticVsInstance obj = new StaticVsInstance();
obj.instanceMethod(); // Call instance method via object
}

static void staticMethod() {
System.out.println("This is a static method.");
}

void instanceMethod() {
System.out.println("This is an instance method.");
}
}

Output

This is a static method.
This is an instance method.

Passing Arrays to Methods

Arrays can be passed to methods to perform operations like summing elements or modifying the array.

public class ArrayMethodExample {
public static void main(String[] args) {
int[] numbers = {1, 2, 3, 4, 5};
int sum = sumArray(numbers);
System.out.println("Sum of array: " + sum);
modifyArray(numbers);
System.out.print("Modified array: ");
for (int num : numbers) {
System.out.print(num + " ");
}
}

static int sumArray(int[] arr) {
int sum = 0;
for (int num : arr) {
sum += num;
}
return sum;
}

static void modifyArray(int[] arr) {
for (int i = 0; i < arr.length; i++) {
arr[i] *= 2; // Double each element
}
}
}

Output

Sum of array: 15
Modified array: 2 4 6 8 10

Recursion

A recursive method calls itself to solve a problem by breaking it into smaller subproblems.

public class RecursionExample {
public static void main(String[] args) {
int result = factorial(5);
System.out.println("Factorial of 5: " + result);
}

static int factorial(int n) {
if (n == 0 || n == 1) {
return 1;
}
return n * factorial(n - 1);
}
}

Output

Factorial of 5: 120

Method Scope and Lifetime

Variables declared inside a method (local variables) have a scope limited to that method and are destroyed when the method ends. Parameters are also local to the method.

public class ScopeExample {
public static void main(String[] args) {
int x = 10; // Local to main
printValue(x);
// System.out.println(localVar); // Error: localVar not accessible
}

static void printValue(int param) {
int localVar = param + 5; // Local to printValue
System.out.println("Local variable: " + localVar);
}
}

Output

Local variable: 15

Example Program with User Input

This program uses methods to calculate the square of a number, check if itâ€™s even, and sum an array, with user input.

import java.util.Scanner;

public class MethodExample {
public static void main(String[] args) {
Scanner input = new Scanner(System.in);
System.out.print("Enter a number: ");
int num = input.nextInt();

int square = calculateSquare(num);
System.out.println("Square of " + num + ": " + square);

boolean isEven = isEven(num);
System.out.println(num + " is even: " + isEven);

System.out.print("Enter size of array: ");
int size = input.nextInt();
int[] array = new int[size];
System.out.println("Enter " + size + " numbers:");
for (int i = 0; i < size; i++) {
System.out.print("Number " + (i + 1) + ": ");
array[i] = input.nextInt();
}
int sum = sumArray(array);
System.out.println("Sum of array: " + sum);

input.close();
}

static int calculateSquare(int number) {
return number * number;
}

static boolean isEven(int number) {
return number % 2 == 0;
}

static int sumArray(int[] arr) {
int sum = 0;
for (int num : arr) {
sum += num;
}
return sum;
}
}

Output

Enter a number: 6
Square of 6: 36
6 is even: true
Enter size of array: 3
Enter 3 numbers:
Number 1: 2
Number 2: 3
Number 3: 4
Sum of array: 9

Important Notes

  • Use static for methods called without creating an object, typically for utility methods.
  • Instance methods require an object and can access instance variables.
  • Recursive methods need a base case to prevent infinite recursion.
  • Method overloading is based on parameter differences, not return type.
  • Method overriding requires inheritance and identical method signatures.
  • Variables declared in a method are local and not accessible outside it.
  • Arrays passed to methods are passed by reference, so modifications affect the original array.
