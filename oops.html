Object-Oriented Programming Concepts in Java

Introduction to OOP

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of objects, which can contain data (fields) and methods. Java is an OOP language that supports four main principles: encapsulation, inheritance, polymorphism, and abstraction.

The Key Concepts in OOP Include:

The following are the core principles of Object-Oriented Programming in Java:

  • Encapsulation: Bundling data and methods within a class, controlling access to data using access modifiers (e.g., private, public) to protect the internal state of an object.
  • Inheritance: Allowing a class to inherit properties and methods from another class, promoting code reuse and establishing a parent-child relationship.
  • Polymorphism: Enabling objects to be treated as instances of their parent class, with the ability to override or overload methods to perform differently based on the object type.
  • Abstraction: Hiding complex implementation details and exposing only the essential features of an object, achieved through abstract classes or interfaces.

Access Specifiers

Access specifiers (or modifiers) control the visibility and accessibility of class members (fields, methods, constructors) in Java.

  • public: Accessible from everywhere.
  • protected: Accessible within the same package and in subclasses (even if in different packages).
  • default (package-private, no modifier): Accessible only within the same package.
  • private: Accessible only within the same class.
public class AccessSpecifierExample {
public int publicVar = 1;
protected int protectedVar = 2;
int defaultVar = 3;
private int privateVar = 4;

public void display() {
System.out.println("Public: " + publicVar);
System.out.println("Protected: " + protectedVar);
System.out.println("Default: " + defaultVar);
System.out.println("Private: " + privateVar);
}
}

public class TestAccess {
public static void main(String[] args) {
AccessSpecifierExample obj = new AccessSpecifierExample();
obj.display();
System.out.println("Accessing publicVar: " + obj.publicVar);
System.out.println("Accessing protectedVar: " + obj.protectedVar);
System.out.println("Accessing defaultVar: " + obj.defaultVar);
// System.out.println(obj.privateVar); // Error: privateVar not accessible
}
}

Output

Public: 1
Protected: 2
Default: 3
Private: 4
Accessing publicVar: 1
Accessing protectedVar: 2
Accessing defaultVar: 3

Classes and Objects

A class is a blueprint for creating objects, defining properties (fields) and behaviors (methods). An object is an instance of a class.

public class Car {
String brand;
int speed;

void display() {
System.out.println("Brand: " + brand + ", Speed: " + speed);
}
}

public class ClassExample {
public static void main(String[] args) {
Car car = new Car(); // Create object
car.brand = "Toyota";
car.speed = 120;
car.display();
}
}

Output

Brand: Toyota, Speed: 120

Encapsulation

Encapsulation involves bundling data (fields) and methods within a class and controlling access using access modifiers (private, public). Getters and setters are used to access private fields.

public class Person {
private String name;
private int age;

// Getter
public String getName() {
return name;
}

// Setter
public void setName(String name) {
this.name = name;
}

public int getAge() {
return age;
}

public void setAge(int age) {
if (age > 0) this.age = age;
}
}

public class EncapsulationExample {
public static void main(String[] args) {
Person person = new Person();
person.setName("Alice");
person.setAge(25);
System.out.println("Name: " + person.getName() + ", Age: " + person.getAge());
}
}

Output

Name: Alice, Age: 25

Inheritance

Inheritance allows a class (subclass) to inherit fields and methods from another class (superclass) using the extends keyword.

class Vehicle {
String type = "Vehicle";
void move() {
System.out.println("The vehicle is moving.");
}
}

class Bike extends Vehicle {
String brand = "Honda";
}

public class InheritanceExample {
public static void main(String[] args) {
Bike bike = new Bike();
System.out.println("Type: " + bike.type + ", Brand: " + bike.brand);
bike.move();
}
}

Output

Type: Vehicle, Brand: Honda
The vehicle is moving.

Types of Inheritance

Java supports the following types of inheritance (note: multiple inheritance of classes is not supported, but can be achieved via interfaces):

  • Single Inheritance: A subclass inherits from one superclass.
  • Multilevel Inheritance: A class inherits from a subclass, forming a chain.
  • Hierarchical Inheritance: Multiple subclasses inherit from a single superclass.

Single Inheritance Example

A subclass inherits from one superclass.

class Animal {
void eat() {
System.out.println("Animal is eating");
}
}

class Dog extends Animal {
void bark() {
System.out.println("Dog is barking");
}
}

public class SingleInheritanceExample {
public static void main(String[] args) {
Dog dog = new Dog();
dog.eat();
dog.bark();
}
}

Output

Animal is eating
Dog is barking

Multilevel Inheritance Example

A class inherits from a subclass, forming a chain.

class Animal {
void eat() {
System.out.println("Animal is eating");
}
}

class Dog extends Animal {
void bark() {
System.out.println("Dog is barking");
}
}

class Puppy extends Dog {
void play() {
System.out.println("Puppy is playing");
}
}

public class MultilevelInheritanceExample {
public static void main(String[] args) {
Puppy puppy = new Puppy();
puppy.eat();
puppy.bark();
puppy.play();
}
}

Output

Animal is eating
Dog is barking
Puppy is playing

Hierarchical Inheritance Example

Multiple subclasses inherit from a single superclass.

class Animal {
void eat() {
System.out.println("Animal is eating");
}
}

class Dog extends Animal {
void bark() {
System.out.println("Dog is barking");
}
}

class Cat extends Animal {
void meow() {
System.out.println("Cat is meowing");
}
}

public class HierarchicalInheritanceExample {
public static void main(String[] args) {
Dog dog = new Dog();
dog.eat();
dog.bark();

Cat cat = new Cat();
cat.eat();
cat.meow();
}
}

Output

Animal is eating
Dog is barking
Animal is eating
Cat is meowing

Polymorphism

Polymorphism allows methods to perform differently based on the object calling them, achieved through method overriding or overloading.

class Animal {
void makeSound() {
System.out.println("Generic animal sound");
}
}

class Dog extends Animal {
@Override
void makeSound() {
System.out.println("Bark");
}
}

public class PolymorphismExample {
public static void main(String[] args) {
Animal animal = new Animal();
Animal dog = new Dog(); // Polymorphism
animal.makeSound();
dog.makeSound();
}
}

Output

Generic animal sound
Bark

Types of Polymorphism

Polymorphism in Java is categorized into two types:

  • Compile-time (Static) Polymorphism: Achieved through method overloading or operator overloading (limited in Java).
  • Runtime (Dynamic) Polymorphism: Achieved through method overriding, where the method called is determined at runtime based on the object type.

Compile-time Polymorphism Example

Method overloading allows multiple methods with the same name but different parameters.

class Calculator {
int add(int a, int b) {
return a + b;
}

double add(double a, double b) {
return a + b;
}
}

public class CompileTimePolymorphismExample {
public static void main(String[] args) {
Calculator calc = new Calculator();
System.out.println("Int sum: " + calc.add(2, 3));
System.out.println("Double sum: " + calc.add(2.5, 3.5));
}
}

Output

Int sum: 5
Double sum: 6.0

Runtime Polymorphism Example

Method overriding allows a subclass to provide a specific implementation of a superclass method.

class Shape {
void draw() {
System.out.println("Drawing a shape");
}
}

class Circle extends Shape {
@Override
void draw() {
System.out.println("Drawing a circle");
}
}

public class RuntimePolymorphismExample {
public static void main(String[] args) {
Shape shape = new Shape();
Shape circle = new Circle();
shape.draw();
circle.draw();
}
}

Output

Drawing a shape
Drawing a circle

Abstraction

Abstraction hides implementation details and shows only essential features, achieved using abstract classes or interfaces.

abstract class Shape {
abstract double calculateArea();
}

class Circle extends Shape {
double radius;

Circle(double radius) {
this.radius = radius;
}

@Override
double calculateArea() {
return Math.PI * radius * radius;
}
}

public class AbstractionExample {
public static void main(String[] args) {
Shape circle = new Circle(5.0);
System.out.printf("Area of circle: %.2f%n", circle.calculateArea());
}
}

Output

Area of circle: 78.54

Example Program with User Input

This program demonstrates OOP concepts, including encapsulation and access specifiers, by creating a class for a student and allowing user input to set and display properties.

import java.util.Scanner;

class Student {
private String name;
private int rollNumber;
private double grade;

public void setName(String name) {
this.name = name;
}

public String getName() {
return name;
}

public void setRollNumber(int rollNumber) {
if (rollNumber > 0) this.rollNumber = rollNumber;
}

public int getRollNumber() {
return rollNumber;
}

public void setGrade(double grade) {
if (grade >= 0 && grade <= 100) this.grade = grade;
}

public double getGrade() {
return grade;
}

public void displayDetails() {
System.out.println("Student: " + name + ", Roll: " + rollNumber + ", Grade: " + grade);
}
}

public class StudentExample {
public static void main(String[] args) {
Scanner input = new Scanner(System.in);

Student student = new Student();

System.out.print("Enter student name: ");
student.setName(input.nextLine());

System.out.print("Enter roll number: ");
student.setRollNumber(input.nextInt());

System.out.print("Enter grade (0-100): ");
student.setGrade(input.nextDouble());

student.displayDetails();

input.close();
}
}

Output

Enter student name: Alice
Enter roll number: 101
Enter grade (0-100): 85.5
Student: Alice, Roll: 101, Grade: 85.5

Important Notes

  • Use private fields with getters and setters for encapsulation.
  • Inheritance promotes code reuse but should not be overused to avoid complex hierarchies.
  • Polymorphism enables flexibility through method overriding (runtime) or overloading (compile-time).
  • Abstract classes cannot be instantiated; they provide a base for subclasses.
  • Interfaces (not shown here) are another way to achieve abstraction, defining methods without implementation.
  • Java does not support multiple inheritance of classes, but interfaces can simulate it.
  • Access specifiers control the visibility of class members, affecting encapsulation and inheritance.
