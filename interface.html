Interfaces in Java

Introduction to Interfaces

An interface in Java is a blueprint of a class that defines a set of abstract methods and constants. It is used to achieve abstraction and support multiple inheritance, as a class can implement multiple interfaces. Interfaces specify what a class must do, but not how to do it.

Key Characteristics of Interfaces

  • Declared using the interface keyword.
  • All methods in an interface are implicitly public and abstract (prior to Java 8).
  • Fields in an interface are implicitly public, static, and final (constants).
  • Classes implement interfaces using the implements keyword.
  • Since Java 8, interfaces can include default and static methods with implementation.

Declaring and Implementing an Interface

An interface is declared with abstract methods, and a class that implements it must provide implementations for all abstract methods.

interface Vehicle {
void start();
void stop();
}

class Car implements Vehicle {
public void start() {
System.out.println("Car is starting");
}

public void stop() {
System.out.println("Car is stopping");
}
}

public class InterfaceExample {
public static void main(String[] args) {
Vehicle car = new Car();
car.start();
car.stop();
}
}

Output

Car is starting
Car is stopping

Default Methods in Interfaces

Since Java 8, interfaces can include default methods with implementation, allowing backward compatibility and shared functionality.

interface Vehicle {
void start();

default void honk() {
System.out.println("Vehicle is honking");
}
}

class Car implements Vehicle {
public void start() {
System.out.println("Car is starting");
}
}

public class DefaultMethodExample {
public static void main(String[] args) {
Vehicle car = new Car();
car.start();
car.honk(); // Calls default method
}
}

Output

Car is starting
Vehicle is honking

Why Use Multiple Inheritance in Interfaces?

Multiple inheritance in interfaces allows a class to inherit behavior from multiple sources, which is not possible with classes in Java due to the diamond problem (ambiguity in method resolution). Interfaces provide a way to achieve this safely because they only define method signatures (or default implementations), avoiding conflicts in state or implementation. Key reasons include:

  • Flexibility: A class can implement multiple interfaces to combine different behaviors, enabling modular design.
  • Contract Enforcement: Interfaces define contracts that a class must follow, ensuring consistent behavior across unrelated classes.
  • Avoiding Class Hierarchy Restrictions: Unlike single inheritance with classes, interfaces allow a class to inherit multiple sets of functionality without being tied to a single superclass.
  • Polymorphism: A class implementing multiple interfaces can be treated as any of those interface types, increasing code reusability.

Multiple Interfaces

A class can implement multiple interfaces, enabling multiple inheritance of behavior in Java.

interface Drivable {
void drive();
}

interface Rechargeable {
void charge();
}

class ElectricCar implements Drivable, Rechargeable {
public void drive() {
System.out.println("Electric car is driving");
}

public void charge() {
System.out.println("Electric car is charging");
}
}

public class MultipleInterfacesExample {
public static void main(String[] args) {
ElectricCar car = new ElectricCar();
car.drive();
car.charge();
}
}

Output

Electric car is driving
Electric car is charging

Multiple Inheritance with Interfaces Example

This example demonstrates multiple inheritance using interfaces to define methods for printing even and odd numbers, with a class implementing both interfaces to provide functionality for printing numbers from 1 to 100.

interface EvenNumberPrinter {
void printEvenNumbers(int start, int end);
}

interface OddNumberPrinter {
void printOddNumbers(int start, int end);
}

class NumberPrinter implements EvenNumberPrinter, OddNumberPrinter {
public void printEvenNumbers(int start, int end) {
System.out.println("Even numbers from " + start + " to " + end + ":");
for (int i = start; i <= end; i++) {
if (i % 2 == 0) {
System.out.print(i + " ");
}
}
System.out.println();
}

public void printOddNumbers(int start, int end) {
System.out.println("Odd numbers from " + start + " to " + end + ":");
for (int i = start; i <= end; i++) {
if (i % 2 != 0) {
System.out.print(i + " ");
}
}
System.out.println();
}
}

public class MultipleInheritanceExample {
public static void main(String[] args) {
NumberPrinter printer = new NumberPrinter();
printer.printEvenNumbers(1, 100);
printer.printOddNumbers(1, 100);
}
}

Output

Example Program: Print Even and Odd Numbers (1 to 100)

This program uses an interface to define methods for printing even and odd numbers from 1 to 100, demonstrating interface implementation with a practical example.

class NumberUtility implements NumberPrinter {
public void printEvenNumbers(int start, int end) {

for (int i = start; i <= end; i++) {
if (i % 2 == 0) {
System.out.print(i + " ");
}
}
System.out.println();
}

public void printOddNumbers(int start, int end) {

for (int i = start; i <= end; i++) {
if (i % 2 != 0) {

}

public class EvenOddPrinter {

Important Notes

  • Interfaces cannot be instantiated; they are implemented by classes.
  • All interface methods are implicitly public and abstract unless specified as default or static.
  • Interfaces support multiple inheritance, unlike classes in Java, allowing a class to implement multiple behaviors.
  • default methods provide a way to add new functionality to interfaces without breaking existing implementations.
  • Interfaces are useful for defining contracts that classes must follow, ensuring consistent behavior.
  • Multiple inheritance with interfaces avoids the diamond problem by not including state or conflicting implementations.
