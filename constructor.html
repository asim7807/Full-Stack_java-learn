Constructors in Java

Introduction to Constructors

A constructor in Java is a special method used to initialize objects of a class. It has the same name as the class, no return type, and is called automatically when an object is created using the new keyword.

Key Characteristics of Constructors

  • Same name as the class.
  • No return type, not even void.
  • Automatically invoked when an object is created.
  • Used to initialize instance variables.
  • Can be overloaded to provide multiple ways to initialize objects.

Types of Constructors

Java supports the following types of constructors:

  • Default Constructor: Provided by Java if no constructor is defined, initializes fields to default values (e.g., 0, null).
  • Parameterized Constructor: Accepts parameters to initialize fields with specific values.
  • Copy Constructor: Initializes an object using another object of the same class.

Default Constructor

A default constructor is automatically provided by Java if no constructor is explicitly defined. It initializes instance variables to their default values.

public class Student {
String name;
int rollNumber;

// Default constructor provided by Java implicitly
void display() {
System.out.println("Name: " + name + ", Roll: " + rollNumber);
}
}

public class DefaultConstructorExample {
public static void main(String[] args) {
Student student = new Student(); // Calls default constructor
student.display();
}
}

Output

Name: null, Roll: 0

Parameterized Constructor

A parameterized constructor accepts parameters to initialize instance variables with specific values during object creation.

public class Student {
String name;
int rollNumber;

// Parameterized constructor
Student(String name, int rollNumber) {
this.name = name;
this.rollNumber = rollNumber;
}

void display() {
System.out.println("Name: " + name + ", Roll: " + rollNumber);
}
}

public class ParameterizedConstructorExample {
public static void main(String[] args) {
Student student = new Student("Alice", 101);
student.display();
}
}

Output

Name: Alice, Roll: 101

Copy Constructor

A copy constructor creates a new object by copying the values of another object of the same class.

public class Student {
String name;
int rollNumber;

// Parameterized constructor
Student(String name, int rollNumber) {
this.name = name;
this.rollNumber = rollNumber;
}

// Copy constructor
Student(Student other) {
this.name = other.name;
this.rollNumber = other.rollNumber;
}

void display() {
System.out.println("Name: " + name + ", Roll: " + rollNumber);
}
}

public class CopyConstructorExample {
public static void main(String[] args) {
Student student1 = new Student("Bob", 102);
Student student2 = new Student(student1); // Copy constructor
student2.display();
}
}

Output

Name: Bob, Roll: 102

Constructor Overloading

Constructor overloading allows multiple constructors in a class with different parameter lists, providing flexibility in object initialization.

public class Student {
String name;
int rollNumber;
double grade;

// Constructor with name and roll number
Student(String name, int rollNumber) {
this.name = name;
this.rollNumber = rollNumber;
}

// Constructor with name, roll number, and grade
Student(String name, int rollNumber, double grade) {
this.name = name;
this.rollNumber = rollNumber;
this.grade = grade;
}

void display() {
System.out.println("Name: " + name + ", Roll: " + rollNumber + ", Grade: " + grade);
}
}

public class ConstructorOverloadingExample {
public static void main(String[] args) {
Student student1 = new Student("Alice", 101);
Student student2 = new Student("Bob", 102, 85.5);
student1.display();
student2.display();
}
}

Output

Name: Alice, Roll: 101, Grade: 0.0
Name: Bob, Roll: 102, Grade: 85.5

Constructor Chaining

Constructor chaining allows one constructor to call another constructor in the same class using this() or in a superclass using super().

class Person {
String name;

Person(String name) {
this.name = name;
}
}

class Student extends Person {
int rollNumber;

// Constructor chaining within the same class
Student(String name) {
this(name, 0); // Calls the parameterized constructor
}

Student(String name, int rollNumber) {
super(name); // Calls superclass constructor
this.rollNumber = rollNumber;
}

void display() {
System.out.println("Name: " + name + ", Roll: " + rollNumber);
}
}

public class ConstructorChainingExample {
public static void main(String[] args) {
Student student1 = new Student("Alice");
Student student2 = new Student("Bob", 102);
student1.display();
student2.display();
}
}

Output

Name: Alice, Roll: 0
Name: Bob, Roll: 102

Example Program with User Input

This program demonstrates constructors by allowing user input to initialize a student object using a parameterized constructor.

import java.util.Scanner;

class Student {
private String name;
private int rollNumber;
private double grade;

// Parameterized constructor
Student(String name, int rollNumber, double grade) {
this.name = name;
this.rollNumber = rollNumber;
if (grade >= 0 && grade <= 100) this.grade = grade;
}

void displayDetails() {
System.out.println("Student: " + name + ", Roll: " + rollNumber + ", Grade: " + grade);
}
}

public class StudentConstructorExample {
public static void main(String[] args) {
Scanner input = new Scanner(System.in);

System.out.print("Enter student name: ");
String name = input.nextLine();

System.out.print("Enter roll number: ");
int rollNumber = input.nextInt();

System.out.print("Enter grade (0-100): ");
double grade = input.nextDouble();

Student student = new Student(name, rollNumber, grade);
student.displayDetails();

input.close();
}
}

Output

Enter student name: Alice
Enter roll number: 101
Enter grade (0-100): 85.5
Student: Alice, Roll: 101, Grade: 85.5

Important Notes

  • Constructors cannot be inherited, but superclass constructors can be called using super().
  • If no constructor is defined, Java provides a default constructor.
  • this() and super() must be the first statement in a constructor.
  • Constructors can be overloaded like methods, allowing multiple initialization options.
  • Copy constructors create a new object with the same values as an existing object.
